========================================
DJANGO BLOG PROJECT FEATURE MAP (FULL)
========================================


0.x → CORE IMPORTS / BASE SYSTEM
--------------------------------
Meaning:
Basic Django modules required to run project

Used in files:
- models.py
- views.py
- urls.py
- forms.py
- admin.py
- apps.py

Examples:
django.shortcuts, django.urls, django.forms


1.x → MAIN MODEL STRUCTURE (POST CORE)
--------------------------------------
Meaning:
Defines blog post structure in database

Includes:
- title
- slug
- author
- body
- publish, created, updated

Used in files:
- models.py (definition)
- admin.py (display)
- templates (list.html, detail.html)
- views.py (data usage)

Purpose:
Represents blog post data


2.x → STATUS SYSTEM (DRAFT / PUBLISHED)
---------------------------------------
Meaning:
Controls visibility of posts

Includes:
- Status class (TextChoices)
- status field

Used in files:
- models.py (definition)
- views.py (filtering only published)
- admin.py (filter/display)

Flow:
Draft → hidden
Published → visible


3.x → TAG SYSTEM (django-taggit)
--------------------------------
Meaning:
Adds categories/labels to posts

Includes:
- TaggableManager
- Tag model
- tag filtering

Used in files:
- models.py (tags field)
- views.py (filter + similar posts)
- urls.py (tag route)
- templates (display tags)
- base.html (latest posts)

Purpose:
Filtering + grouping + SEO


4.x → MANAGERS (QUERY CONTROL)
------------------------------
Meaning:
Controls how data is fetched

Includes:
- objects manager
- published manager

Used in files:
- models.py (definition)
- views.py (Post.published.all())

Purpose:
Simplifies queries


5.x → META CONFIGURATION
------------------------
Meaning:
Controls DB behavior

Includes:
- ordering
- indexes

Used in files:
- models.py

Purpose:
Performance + default sorting


6.x → STRING REPRESENTATION
---------------------------
Meaning:
Controls how object appears in admin

Includes:
- __str__()

Used in files:

Purpose:
Readable admin output


7.x → ABSOLUTE URL SYSTEM
-------------------------
Meaning:
Generates dynamic URLs

Includes:
- get_absolute_url()
- reverse()
- URL patterns

Used in files:
- models.py (function)
- views.py (email + templates)
- urls.py (pattern)
- templates (links)

Flow:
Model → URL → Template


8.x → COMMENT SYSTEM (FULL FLOW)
--------------------------------
Meaning:
Handles user comments

Includes:
- Comment model
- CommentForm
- comment view
- templates

Used in files:
- models.py (Comment model)
- forms.py (CommentForm)
- views.py (post_comment)
- urls.py (comment route)
- templates (comment.html, comment_form.html, detail.html)
- admin.py (CommentAdmin)

Flow:
User → form → POST → save → display


9.x → PAGINATION SYSTEM
-----------------------
Meaning:
Splits posts into pages

Includes:
- Paginator

- pagination UI

Used in files:
- views.py (logic)

Purpose:
Better UX + performance

----------------------------------
Meaning:

Used in files:

Content discovery


11.x → EMAIL SYSTEM (SHARING POSTS)
-----------------------------------
Meaning:

Used in files:
- forms.py

- urls.py
- templates (share.html)

Flow:
User → form → validate → send email
